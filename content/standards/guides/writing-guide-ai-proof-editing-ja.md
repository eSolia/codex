# ライティングガイド：AI臭を消す文章編集術

> eSolia INTERNAL — Not for distribution outside eSolia
>
> **関連ガイド:** [English version](WRITING_GUIDE_AI_PROOF_EDITING_EN.md) | [Article Writing Guide](ARTICLE_WRITING_GUIDE.md) | [Content Localization Strategy](CONTENT_LOCALIZATION_STRATEGY.md)

## 目的

本ガイドは、イソリアのチームメンバーがAI生成テキスト特有のパターンを見抜き、修正するためのものです。提案書、クライアント向けメール、技術文書、ブログ記事など、あらゆる文書作成・編集時にチェックリストとして活用してください。

AI生成の日本語には、英語とは異なる独自の「癖」があります。英語のAI文章が工業製品のように均一なら、日本語のAI文章は「敬語のコスプレ」です。表面上は丁寧で整っているのに、読み手に響かない。本ガイドでは、その原因と対策を具体的に解説します。

---

## 本ガイドの使い方

2つの場面で活用してください。

1. **執筆時のチェックリストとして。** クライアント向け文書や公開コンテンツを最終確認する際、本ガイドのパターンに該当する箇所がないか確認し、修正する。
2. **編集・レビュー時の参考資料として。** 同僚のドラフトやAIで作成した下書きをレビューする際、不自然な箇所を特定するために使う。

---

## 検出：語彙の危険信号

AI生成の日本語には、異常な頻度で登場する単語や表現があります。一つひとつは正しい日本語ですが、密度が問題です。1段落に3つ以上該当したら書き直しのサインです。

### 多用される単語

| 要注意ワード                 | なぜ危険か                                                     |
| ---------------------------- | -------------------------------------------------------------- |
| 重要、不可欠、必要不可欠     | AIの万能強調語。具体的に何が重要かを書けば、この語は不要になる |
| 様々な、多岐にわたる、幅広い | 「たくさん」と言いたいだけ。具体例を挙げる方が説得力がある     |
| 包括的、網羅的、体系的       | 内容が伴わなければただの飾り                                   |
| 効率的、効果的、最適化       | ビジネス文書の「塩コショウ」。使いすぎると何も味がしなくなる   |
| 促進する、推進する、醸成する | 主語が曖昧なまま使われがち。「誰が」「何を」が消える           |
| 活用、連携、取り組み         | 公的文書のコピペ感が出る                                       |

### 多用されるフレーズ

| フレーズ                                         | 対処法                                                     |
| ------------------------------------------------ | ---------------------------------------------------------- |
| 「～と言えるでしょう」                           | 言い切る。「～です」で十分。                               |
| 「～することができます」                         | 「～できます」に短縮。ほぼ毎回これで済む。                 |
| 「～において」「～における」                     | 「～で」「～の」に置き換え可能なケースが大半。             |
| 「～だけでなく～も」                             | 1回なら自然。2回以上は構文の使い回し。                     |
| 「～について詳しく見ていきましょう」             | 本文に入る。読者は説明が始まることを理解している。         |
| 「～が求められています」                         | 誰が求めているのか主語を明記する。                         |
| 「～を実現します」                               | 何がどう変わるのか具体的に書く。                           |
| 「いかがでしたか？」「いかがでしたでしょうか？」 | ブログ記事AIの代名詞。削除一択。                           |
| 「まず～、次に～、最後に～」                     | たまにはいいが、毎回この構造だとAI臭が強い。               |
| 「～していきましょう」                           | 馴れ馴れしい。相手との距離感を考えて使う。                 |
| 「重要なポイント」「重要な点」                   | 見出しの常套句。内容を具体的に示す見出しに変える。         |
| 「効果的な～」                                   | 何が効果的かを示す方が伝わる。形容詞を削って具体化。       |
| 「～の実態」「本当の～」「実際の～」             | 「実は」系の前振り。英語の"actually"と同じ問題。直接書く。 |

## 検出：構造パターン

語彙よりも、文章の「型」の方がAI検出の精度は高いです。

### 前置きの長さ

AIは本題に入るまでが長い。「近年、デジタルトランスフォーメーションの進展に伴い、企業を取り巻く環境は大きく変化しています。そのような中で～」のような導入が2〜3文続いたら要注意です。

**AI的：** 「近年、IT環境の複雑化が進む中、セキュリティ対策の重要性はますます高まっています。このような状況において、ゼロトラストアーキテクチャが注目を集めています。本稿では～」

**人間的：** 「境界型防御はもう限界です。ゼロトラストへの移行方法を説明します。」

### 「まとめ」の強迫観念

AIは必ず文末にまとめを入れたがります。300字のメールにも「まとめ」セクションを付けることがあります。読み手が直前に読んだ内容を要約する必要はありません。最後の要点を述べたら、そこで終わる。

### 均一な段落長

AI生成の日本語は段落の長さが驚くほど揃います。すべて3〜4文で、同じようなリズム。人間の文章は不均一です。1文だけの段落があっていい。7文の段落があってもいい。

### 過剰な接続詞

「また」「さらに」「加えて」「一方で」「そのため」「したがって」が連打される文章はAIの典型です。すべての文が接続詞で始まる段落を見つけたら、半分以上を削除しても意味は通じるはずです。

### 対称構造への執着

AIは対句やバランスの取れた構造を好みます。「AだけでなくBも」「CとDの両立」「Eの向上とFの改善」が同じ文書に何度も出現したら、パターン化しています。

### 見出しの常套化

AIは「～とは」「～の実態」「～が重要な理由」「～のポイント」のような汎用見出しを好みます。どの記事にも使い回せる見出しはAIの痕跡です。その節の内容を具体的に表す見出しに変える。「重要なポイント」→「押さえておくべきこと」や、内容そのものを見出しにする。

## 検出：トーンの問題

### 敬語の過剰装備

AI生成の日本語は必要以上に丁寧になりがちです。「ご確認いただけますと幸いです」「何卒よろしくお願い申し上げます」が文書全体に散りばめられ、慇懃無礼（いんぎんぶれい）の領域に入ることがあります。

クライアントとの関係性に応じて敬語レベルを調整してください。長年の付き合いがある相手に、初対面レベルの敬語で書くのは不自然です。

### 主語の消失

日本語は主語を省略できる言語ですが、AIはこれを悪用します。「実現できます」「最適化されます」「期待されます」と書き続け、最終的に誰が何をするのか読み手に分からなくなる。ビジネス文書では主語を意識的に明示してください。

### 断言回避

AIは「～と考えられます」「～かもしれません」「～の可能性があります」を多用して、あらゆる主張を曖昧にします。提案書で自社のサービスを勧めるのに「効果が期待できるかもしれません」では弱すぎます。

自信のある主張は言い切る。不確実な部分だけを正直に留保する。

## 修正テクニック

### 具体性を注入する

抽象表現を具体的な数字・名前・事実に置き換えるのが最速の修正法です。

**修正前（AI的）：** 「本ソリューションにより、業務プロセスの効率化と生産性の向上が期待できます。」

**修正後（人間的）：** 「請求書処理が3日から4時間に短縮されます。」

具体的な情報はAIが持っていないため、差し替えるだけで一気に人間らしくなります。

### 文の長さにリズムをつける

AIの日本語は中程度の長さの文が続きます。意識的に短文と長文を混ぜてください。

短い文は目を引く。一方、少し長めに展開して、途中で補足を挟みながら着地する文も、読み手にとっては自然なリズムです。音読して単調に聞こえたら、構造を崩してください。

### 言い切る

AIは保険をかけるように訓練されています。あなたはそうではありません。「テナント統合を推奨します」は「テナント統合も選択肢の一つとして検討に値するかもしれません」より格段に強く、読み手に伝わります。

トレードオフは認めつつ、立場は明確に。

### 話し言葉の要素を入れる

あまりにも書き言葉に寄りすぎた文章はAI臭がします。文脈に応じて、ビジネス上許容される範囲で話し言葉的な表現を混ぜると自然になります。

- 「～なんですが」（説明の柔らかい導入）
- 「正直なところ」（率直さの表現）
- 「ざっくり言えば」（要約の自然な前置き）

ただし、クライアントへの正式な提案書では控えめに。社内文書やメールでは積極的に。

### 一文目を削る

段落ごとに最初の一文を消してみてください。それでも意味が通じるなら（たいてい通じます）、その一文はAI的な前置きです。この一手間で文章の密度が上がります。

### 「まとめ」を削る

最終段落が「以上のことから」「まとめると」「総括すると」で始まっていたら、その段落を削除してください。文書が明確なら、読者に再度要約する必要はありません。

## クイックリファレンス・チェックリスト

クライアント向け・公開文書を送信する前に確認してください。

- [ ] 「要注意ワード」表の語句が1段落に3つ以上集中していないか
- [ ] 前置きが2文以上続いていないか（本題に早く入る）
- [ ] 「～することができます」を「～できます」に短縮したか
- [ ] 主語が明確か（「誰が」「何が」が分かるか）
- [ ] 各主張に具体的な数字・事実・事例があるか
- [ ] 段落の長さにバラつきがあるか（均一は不自然）
- [ ] 文末表現が単調でないか（「～です」「～ます」の連打を避ける）
- [ ] 過剰な接続詞を削除したか
- [ ] 不要な「まとめ」セクションを削除したか
- [ ] 見出しが汎用的すぎないか（「重要なポイント」「効果的な方法」は具体化する）
- [ ] 音読して、イソリアの人間が書いた文章に聞こえるか

## AI活用についての補足

本ガイドはAI利用を否定するものではありません。ClaudeなどのAIツールを下書きやアイデア出しに使うのは効率的であり、推奨されています。

重要なのは、AI生成テキストはあくまで「素材」であり、完成品ではないということです。あなたの専門知識、意見、クライアント固有の状況に関する理解を反映させる編集工程こそが、汎用的な出力をイソリアの文書に変える作業です。その部分はモデルにはできません。
